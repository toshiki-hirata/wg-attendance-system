# プレゼン資料作成ルール

このドキュメントは、Vueハンズオン用のプレゼン資料作成に関するルールとガイドラインを定めています。

## 使用ツール

**Marp (Markdown Presentation Ecosystem)** を使用します。

### Marpを選択した理由
- Markdownベースでバージョン管理が容易
- VS Code拡張機能で編集とプレビューが可能
- コードハイライトが美しく、技術プレゼンに最適
- PDFやHTMLへのエクスポートが簡単
- GitHubでの管理と共有が容易

## ディレクトリ構造

```
hands_on_docs/
├── PRESENTATION_RULES.md    # このファイル
├── slides/                  # プレゼンテーションファイル
│   ├── 01_introduction.md   # イントロダクション（Vueの基本座学）、環境準備（Stack blitzを利用）
│   ├── 02_vue_basics.md     # Vueアプリ全体の構成、HTML、見た目の実装

├── assets/                  # 画像やその他のアセット
│   ├── images/

```

## スライド作成ルール

### 1. ファイル命名規則
- `番号_内容.md` 形式を使用（例：`01_introduction.md`）
- 番号は2桁のゼロパディング
- 英語のスネークケースを使用

### 2. スライド構成

#### 前提
- 1つのMarkdownファイルに1つのトピック
- `---` で各スライドを区切る
- 最初のスライドにはセクションタイトルを含める

#### 共通アジェンダ構成
トピックに対する説明をした後、本アプリのソースコードベースに問題を解いてもらう形式とする。

1. その日使う資源共有
2. トピックの座学（ストーリー性を持たせて構成）
3. 問題初級（選択肢みて回答）
4. 問題中級（お題に対して、実際にソースをいじる）＝ハンズオン
5. 解説

**ハンズオンの実施形式**:
- 参加者には**未完成状態のコード**を事前に配布
- 座学で学んだ内容を活用して、問題（中級）で**完成状態**に実装
- 完成系のコードは本リポジトリで管理し、解説時に参考として提示

**座学セクションの構成例**:
- 全体像の提示（例：.vueファイルには3つのセクションがある）
- 各要素の詳細説明（例：①template → ②script → ③style）
- 要素間の関連性の説明（例：scriptで定義したデータをtemplateで使用）
- 実践的な活用方法（例：styleの代わりにTailwind CSSを使う方法）

**問題設計の指針**:
- 初級問題：基本概念の理解確認（例：「○○するにはどのコードが適切？」形式）
- 中級問題：配布された未完成コードを完成させる実装課題（ハンズオン形式）
- 座学内容と問題の連携：学んだ内容を直接活用できる問題設計

**コード管理の方針**:
- **完成系コード**: 本リポジトリで管理（解説・参考用）
- **未完成コード**: 参加者配布用（StackBlitzまたは別リポジトリで管理）
- **実装目標**: 未完成→完成への明確な変化を設定



### 3. スタイルガイド

#### ストーリー性のある構成
- **各セクション間の繋がりを重視**する
- 前のセクションで説明した内容を次のセクションで活用する
- 唐突な話題転換を避け、自然な流れを作る
- 例：
  - .vueファイルの構成（template, script, style）を説明
  - → 各セクションの詳細を順番に解説
  - → styleセクションからTailwind CSSへ自然に移行

#### 初学者への配慮
- 専門用語は初出時に必ず説明する
- 英語の略語（例：SFC）より日本語の説明を優先
- 具体例を多用し、抽象的な説明を避ける
- 「なぜそうするのか」を明確にする

#### 日本語表記
- 技術用語は英語表記を基本とする
- カタカナ表記は一般的なもののみ使用
- 例：
  - ⭕ Component、Store、State
  - ⭕ コンポーネント、ストア（一般的な場合）
  - ❌ ステート（Stateを使用）

#### スライドデザイン
- 1スライド1トピックを原則とする
- 箇条書きは3〜5項目まで
- コードサンプルは実行可能な最小限のものを使用
- 重要なポイントは **太字** で強調

#### コードブロック
<pre>
```markdown
```typescript
// TypeScriptコードの例
const message: string = 'Hello Vue!';
```
```
</pre>

- 言語指定を必ず行う（typescript, vue, bash など）
- 実際のプロジェクトコードを参照する場合は、ファイルパスをコメントで明記

#### 画像の扱い

```markdown
![説明テキスト](../assets/images/画像名.png)
```
- 相対パスで参照
- 画像は `assets/images/` に配置
- 適切なalt textを必ず含める

### 4. ハンズオン用スライドの特別ルール

#### 問題スライドのフォーマット

- サンプル問題: @./SAMPLE_PROBLEM.md

##### 初級問題（選択式）
```markdown
## 🔍 問題（初級）

**Q: [質問内容]**

A. 選択肢1
B. 選択肢2
C. 選択肢3
D. 選択肢4

⏰ 考え時間: 1分
```

##### 中級問題（ハンズオン）
```markdown
## 🔨 問題（中級）- ハンズオン

**お題**: [未完成状態から完成状態への実装課題]

**現在の状況**:
- [渡されたコードの未完成状態を説明]
- [具体的な問題点を明記]

**要件**:
- 要件1
- 要件2
- 要件3

**完成目標**:
[完成した時の動作や表示を明確に示す]

**ヒント**: 
- 該当ファイル: `src/components/example.vue`
- 今日学んだ内容との関連性を明示（例：「3つのセクションをすべて活用」）

⏰ 実装時間: 10分
```

**中級問題の設計原則**:
- **事前配布**: 参加者には未完成状態のコードを配布
- **実装課題**: 座学で学んだ内容を活用して完成状態に実装
- **明確なゴール**: 「動かない→動く」「表示されない→表示される」など分かりやすい変化
- **段階的実装**: 学習内容を総合的に活用する課題設定
- **達成感**: 参加者が「作った！」と実感できる具体的な成果物

#### 解説スライド
```markdown
## 💡 解説

**正解**: [初級問題の場合は選択肢]

**ポイント**:
- 重要な概念の説明
- なぜこのアプローチが良いのか
- 実務での応用例

```[言語]
// 解答例のコード
```
```

#### 資源共有スライド
```markdown
## 📚 本日の資源

### リポジトリ
- メインリポジトリ: [URL]
- StackBlitz環境: [URL]

### 参考資料
- Vue.js公式ドキュメント: [該当セクション]
- 追加資料: [あれば]

### 今日のゴール
✅ [達成目標1]
✅ [達成目標2]
✅ [達成目標3]
```

### 5. StackBlitz環境の利用

#### 環境準備スライド
```markdown
## 🚀 環境準備

### StackBlitzでの開始方法
1. 提供されたURL（未完成版）にアクセス
2. 「Fork」ボタンをクリック
3. 自分の環境が立ち上がることを確認

### 動作確認
- ターミナルで `npm run dev` が実行されているか確認
- プレビュー画面でアプリが表示されているか確認
- **重要**: 未完成状態であることを確認（例：時刻が「--:--:--」表示）
```

#### ハンズオン用の環境管理
- **完成版**: 本リポジトリで管理（講師・解説用）
- **未完成版**: StackBlitzで別途作成・配布（参加者用）
- **配布方法**: StackBlitzのテンプレート機能またはGitHubの別ブランチを利用

#### 注意事項
- StackBlitzの制限事項を明記する
- オフライン時の代替手段を準備
- ローカル環境セットアップ手順も別途用意

## VS Code拡張機能の設定

必須の拡張機能：
- **Marp for VS Code** (marp-team.marp-vscode)

推奨設定（`.vscode/settings.json`）：
```json
{
  "markdown.marp.enableHtml": true,
  "markdown.marp.exportType": "pdf",
  "markdown.marp.chromePath": ""
}
```

## エクスポート手順

### PDFエクスポート
1. VS CodeでMarkdownファイルを開く
2. コマンドパレット（Cmd+Shift+P）で「Marp: Export Slide Deck...」を選択
3. PDFを選択し、`exports/` ディレクトリに保存

### HTMLエクスポート
- 配布用にはHTMLエクスポートを推奨
- 参加者がブラウザで閲覧可能

## プレゼンテーション実施時の注意

1. **事前準備**
   - PDFとHTMLの両方をエクスポートしておく
   - オフラインでも表示できるように準備

2. **画面共有時**
   - VS Codeのプレビューモードを使用
   - フォントサイズは見やすいように調整

3. **コードデモ**
   - 実際のプロジェクトで動作確認
   - エラーハンドリングの例も準備
